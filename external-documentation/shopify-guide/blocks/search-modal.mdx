</Info>

The search modal is what shoppers use to run a search and get to the results page. You need to give them a way to open it.

There are two ways to do this.

<AccordionGroup>
  <Accordion title="Option 1: Re-use your existing search button">
    <Steps>
      <Step title="Remove the old behaviour">
        Remove any event listeners from your existing search button so nothing happens when it's clicked.
      </Step>
      <Step title="Add the class">
        Add the `open-depict-search` class to your search button. Depict attaches a click handler to any element with this class.
      </Step>
      <Step title="Done">
        Clicking the button now opens the Depict search modal.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Option 2: Add the Depict Search Field block">
    The **Depict Search Field** block can render either a visible search field or an icon-only button, and it connects to the modal automatically. It's available both as a section block and a header block (both appear as "Depict Search Field").

    <Steps>
      <Step title="Add the block">
        In the theme editor, add a **Depict Search Field** block where you want it — commonly the header.

        <Frame caption="Adding a search field block in the theme editor" style={{ maxWidth:"500px" }}>
          <img
            src="https://cdn.statically.io/gh/depict-org/documentation-images/main/shopify/adding-search-field.svg"
            style={{aspectRatio: 1280/2144, width: "100%"}}
            ref={element => {
          // In prod, they have some zoom thing which breaks aspectRatio, work around that. Unfortunately this is not in the SSR'd output so there's still some CLS.
          const direct_parent = element?.parentElement;
          if(direct_parent?.matches?.("[data-rmiz-content]")) {
              Object.assign(direct_parent.style, {
                  width: "100%",
                  display: "flex",
                  justifyContent: "center",
                  alignItems: "center"
              })
          }
          const parents_parent = direct_parent?.parentElement;
          if(parents_parent?.matches?.("[aria-owns^='rmiz-modal']")) {
              parents_parent.style.width = "100%";
          }
        }}
          />
        </Frame>
      </Step>
      <Step title="Choose the layout">
        In the block settings, set **Layout** to:

        - **Search field** — a visible search field in your header.
        - **Icon only** — just a button (the icon-only option that replaces the legacy "Depict Search Button" block).

        Use **Alignment** (Left / Center / Right) to position the block.
      </Step>
      <Step title="Done">
        You now have a search field (or icon) that opens the Depict search modal, aligned to it.

        <Frame caption='The "Depict Search Field" block'>
          <img

          />
        </Frame>

        <Frame caption="The search modal aligning to the search field when focusing/clicking it">
          <img

## Optional: Alignment options

When using the **search field** layout, the modal automatically aligns to the field (landscape modal), or aligns to it initially and then animates horizontally to be centered (portrait modal).

When using the **icon-only** layout or a re-used button, there are two alignment modes available, provided you're using the landscape search modal:

### Enabling the vertically-aligned-to-button mode

<Steps>
  <Step title="Depict Search Field block (Icon only)">
    Set **Layout** to **Icon only**, then set **Modal when opened aligns to** to **Search field**. (Choose **Page center** for the centered default.)
  </Step>
  <Step title="Re-used search button">
    Add `data-align-to-self="true"` to your button element.
  </Step>
</Steps>

<Info>
  Let us know if you'd like to align the modal to other, arbitrary DOM elements.
</Info>
